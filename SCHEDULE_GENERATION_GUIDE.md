# Schedule Generation Guide

## Как подготовить данные перед генерацией

Чтобы расписание сгенерировалось без конфликтов, администратор должен заполнять сущности в таком порядке:

1. `Teachers`
2. `Rooms`
3. `Groups`
4. `Disciplines`
5. `Sections`
6. Генерация расписания

## Что означают сущности

### Disciplines
Это сами предметы:
- кто ведет
- на каком курсе
- в каком семестре
- нужны ли компьютеры

### Sections
Это привязка предмета к группе:
- для какой группы идет предмет
- сколько занятий в неделю нужно поставить
- какой тип занятия используется

Если `sections` не заполнены, расписание не создастся.

## Что обязательно проверить

### Teachers
- задан преподаватель
- указан факультет
- указаны языки преподавания

### Rooms
- указана вместимость
- указан тип аудитории
- указано количество компьютеров
- аудитория доступна

### Groups
- указан курс обучения группы
- указан язык обучения
- указано число студентов
- отмечено, есть ли подгруппы

### Disciplines
- назначен преподаватель
- указан курс
- указан семестр
- если предмет компьютерный, включено `requires_computers`

### Sections
- выбран существующий предмет
- выбрана существующая группа
- указано число занятий в неделю
- указан тип занятия

## Что учитывает генератор

- язык группы
- языки преподавателя
- вместимость аудитории
- тип аудитории
- количество компьютеров
- подгруппы
- предпочтения преподавателей, если они одобрены

## Когда генерация падает

Обычно причина одна из этих:

- мало данных
- нет секций
- преподаватель не подходит по языку
- не хватает подходящих аудиторий
- не хватает компьютерных кабинетов
- вместимость кабинета меньше размера группы

## Практический порядок для админа

1. Заполнить преподавателей
2. Заполнить аудитории
3. Заполнить группы
4. Заполнить дисциплины
5. Заполнить секции
6. Открыть страницу расписания
7. Выбрать семестр и учебный год
8. Нажать генерацию
