## 🔄 Жизненный цикл

1. **При загрузке приложения** → проверяем `localStorage` на наличие `user`
2. **При клике "Логин"** → перейти на `/login`
3. **При клике "Регистрация"** → перейти на `/register`
4. **После успешного входа/регистрации**:
   - Сохраняем данные пользователя (без пароля) в `localStorage`
   - Перенаправляем на `/` (главная)
   - Показываем имя пользователя и кнопку `Logout` в хедере
5. **При клике "Выход"** → удаляем данные, перенаправляем на `/login`

## 📡 Подключение реального API

Когда бэкенд будет готов, замените функции в `src/services/mockData.js`:

```javascript
// Было:
export const mockAuthAPI = {
  login: async (email, password) => { ... }
}

// Станет:
export const mockAuthAPI = {
  login: async (email, password) => {
    const response = await fetch('http://backend.com/api/auth/login', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ email, password })
    });
    return response.json();
  }
}
```

## ⚠️ Важно

- Пароли хранятся **только в памяти** (mockData)
- При перезагрузке страницы новые регистрации **теряются**
- В БД/реальном бэкенде пароли должны быть **хеш-защищены** (bcrypt, argon2)
- Токены передаются в `localStorage` (в production использовать httpOnly cookies)
